# gap-detector v5 (post-G3) — gaps in /god-review pipeline

### [high] Codex-validation gating prose says "round % CODEX_VALIDATION_EVERY == 0" but no code implements the gate
**Source:** gap-detector
**Evidence:** ~/.claude-dotfiles/commands/god-review.md:672 (prose); 698-701 (the bash that invokes codex-invoke.sh has no `if (( ROUND % CODEX_VALIDATION_EVERY == 0 ))` guard)
**Impact:** `--codex-validation-every N` flag is parsed (line 64-65) and validated (≥1) but never honored. Codex validation runs every round regardless. The "(unverified-this-round)" tagging branch the prose describes never fires.
**Fix:** Wrap the codex-invoke.sh call (lines 698-701) and the Codex-finds Claude-validates Agent call (line ~706) in `if [ "$CODEX_AVAILABLE" = "true" ] && [ $((ROUND % CODEX_VALIDATION_EVERY)) -eq 0 ]; then ... else echo "(skipped: round $ROUND % $CODEX_VALIDATION_EVERY != 0)"; fi`. Also instruct orchestrator to tag findings `(unverified-this-round)` in the else branch.

### [high] FROZEN_UNITS_CAP declared and exported but never compared anywhere
**Source:** gap-detector
**Evidence:** god-review.md:158 declares `FROZEN_UNITS_CAP="${FROZEN_UNITS_CAP:-3}"`; env-helpers.sh:36 omits it from write_env list (so it doesn't even survive cross-block); only references are init at 158 and unused FROZEN_UNITS_COUNT counter at 869. No `[ "$FROZEN_UNITS_COUNT" -ge "$FROZEN_UNITS_CAP" ]` check, no abort, no termination.
**Impact:** "Bounded-mode freeze cap" is documented in the Tunable Constants block but spec→code dead-end. Frozen units accumulate without bound. Indistinguishable from a runaway churn loop except via the separate INSTABILITY_RATE backstop.
**Fix:** After `record_frozen` or in 3g backstops, add `if [ "$(python3 -c "import json;print(len(json.load(open('$WORKDIR/tmp/god-review/state.json'))['frozen_units']))")" -ge "$FROZEN_UNITS_CAP" ]; then echo "Frozen-units cap reached"; LOOP_EXIT=frozen_cap; fi`. Add FROZEN_UNITS_CAP to env-helpers.sh:36 export list.

### [high] Codex validation output file (`/tmp/codex-validation-output.txt`) is written but never read/parsed
**Source:** gap-detector
**Evidence:** god-review.md:699 writes to `/tmp/codex-validation-output.txt` via codex-invoke.sh. No subsequent grep/parse/read of that file anywhere in god-review.md (verified with grep).
**Impact:** Codex's CONFIRMED/FALSE_POSITIVE/UNCERTAIN verdicts on Claude-found findings are computed but discarded. The `record_false_positive` call documented at 718-722 has no upstream caller — it's a helper with prose telling the orchestrator to call it but no concrete bash that reads the Codex output file and routes verdicts. Phase 2e promotion logic at 743 ("CONFIRMED by validator → promote") relies on signal that's never extracted.
**Fix:** After line 701, add a parse block: `python3 -c '...read /tmp/codex-validation-output.txt, regex "FINDING_ID: STATUS — reason", emit FP entries via record_false_positive, emit CONFIRMED list to /tmp/codex-confirmed.txt'`. Then have the orchestrator's promotion pass at 2e read /tmp/codex-confirmed.txt.

### [medium] `--rounds N` flag in /god-report parsed and validated but report-merge logic is prose-only
**Source:** gap-detector
**Evidence:** god-report.md:38-40 parses `--rounds`. Line 95-100 prose says "After Phase 2 completes, repeat Phase 0–2 (ROUNDS-1) more times. Each round writes a separate tmp/god-review/report-round-N.md ... merge ... union-deduplicating findings by hash". No bash, no concrete file naming, no merge code, no `(consistent across rounds)` tagging logic. Phase 0/1/2 in god-review.md unconditionally writes `tmp/god-review/report.md` — running it N times overwrites the same path.
**Impact:** `/god-report --rounds 3` runs Phase 0–2 once (because god-review.md doesn't loop) and writes report.md once. The N-round de-noising feature is unimplemented prose.
**Fix:** Either implement the merge in god-report.md as concrete bash (rename report.md→report-round-N.md after each pass; final python3 merge step computing hash-keyed dict + `(consistent across rounds: K/N)` tag), or document that `--rounds` is a no-op pending implementation.

### [medium] Phase 2c "ACTIVE_PRINCIPLES = ALWAYS_ON + stack-gated" lives in pseudocode block, no concrete bash builds the list
**Source:** gap-detector
**Evidence:** god-review.md:452-487 declares ACTIVE_PRINCIPLES in a fenced ``` (no `bash` lang tag) pseudocode block. The orchestrator is expected to "spawn one Agent per principle in ACTIVE_PRINCIPLES" but there is no bash array, no `for principle in ${ACTIVE_PRINCIPLES[@]}`, no write_env export. The agent count claim ("23 × 2") relies on the orchestrator interpreting prose.
**Impact:** Whether stack-gated principles activate depends entirely on the orchestrator's prose-following discipline. No mechanical guarantee architecture-backend / architecture-frontend / self-contained / tanstack-query / perf-benchmark are skipped when their HAS_* signal is empty. **Fix:** Add a concrete bash block that emits `ACTIVE_PRINCIPLES=(...)` based on HAS_* env vars, writes the list to `$WORKDIR/tmp/god-review/active-principles.txt`, and have Phase 2c spawn-instructions reference that file.

### [medium] `human_gate_emitted[].round` field written but never read for "still pending as of round N" tagging

### [medium] /god-report Phase 0/1/2 delegation says "execute the contents of god-review.md" but god-review.md Phase 1b unconditionally creates a snapshot ref + state.json
**Source:** gap-detector
**Evidence:** god-report.md:80-94 instructs orchestrator to execute god-review.md Phase 0/1/2. god-review.md:362-416 (Phase 1b) creates snapshot refs and writes `tmp/god-review/state.json` with full Phase-3 schema (round, frozen_units, kept_fixes, reverted_fixes, etc.). For /god-report, Phase 3 never runs — but the snapshot+state.json are still emitted.
**Impact:** /god-report leaves a misleading state.json on disk that looks like a stale Phase-3 run. Next `/god-review --resume` could pick it up. The git stash created at Phase 1b leaks into stash list with no consumer.
**Fix:** Either (a) make Phase 1b conditional on a `WILL_FIX` flag set by /god-review only, or (b) have /god-report append a final cleanup block that removes state.json and drops the unused stash. Document explicitly that snapshotting is /god-review's domain.
