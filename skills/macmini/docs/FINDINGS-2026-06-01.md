## The "Send system keys" dependency (load-bearing for ALL Cmd niceties)

`Cmd+Space` / `Cmd+Tab` / `Cmd+C` / `Cmd+V` forward to the mini **ONLY when
CRD's "Send system keys" toggle is ON.** That's a per-profile USER gesture the
agent **cannot set** — this session confirmed CRD's a11y tree exposes ONLY the
`Desktop` textbox (no panel uids). The live Cmd+Space test worked because the
toggle was already on.

Consequences baked into the skill:
- `setup.md` / `connect.md` keep the one-time "enable Send system keys +
  Synchronize clipboard" instruction (user-only, persists).
- Every Cmd nicety has an explicit **click fallback** (Dock icon / Apple menu).
  If `Meta+Space` does nothing -> toggle off -> surface to the user.
- `connect.md`'s `wait_for(["Send system keys","Synchronize clipboard"])` mount
  signal doubles as a "the toggle exists in this panel" check.

## macOS scroll keys (carried forward, still true)

macOS uses **overlay scrollbars** (no persistent arrow buttons) and reads a
`drag` as text-selection. **Scroll is KEYBOARD only — never `drag` to scroll.**
Click into the content pane to focus it first, then:

| To do this | Press |
|---|---|
| Scroll one screenful down | `press_key("PageDown")` or `"Space"` |
| Scroll one screenful up | `press_key("PageUp")` or `"Shift+Space"` |
| Scroll one line | `press_key("ArrowDown")` / `"ArrowUp"` |
| Jump to bottom | `press_key("End")` or `"Meta+ArrowDown"` |
| Jump to top | `press_key("Home")` or `"Meta+ArrowUp"` |

Reading long output: `PageDown` x N with a screenshot between presses; stitch
top-to-bottom; **return focus to the live tail** (`End`) before the next
keystroke, or it lands in scrollback and is lost.

## Coordinate model (the math, restated)

- Host streams `1920x1080` into a `<canvas>`; rect `{x:9.66,y:0,w:1690.66,h:951}`.
- `getBoundingClientRect()` and `click_at` are **both CSS px** -> rect-derived
  click coords are **NOT** /DPR.
- Only a target eyeballed off a raw screenshot is /DPR (screenshots save at
  `innerW x DPR`), and that path must also subtract the canvas letterbox offset
  (`rect.x`/`rect.y`).
- Read DPR / host size live via the rect helper; never hardcode `2`, `1920`,
  `1080`, or the ephemeral session ID — bind by title (`plan2bid-minim4`).

## Credential safety (the `--secure` story, preserved without gist)

The gist-era `paste --secure` never sent the secret to GitHub — it shipped a
`read -s` bootstrap. The principle survives the gist removal:

- **To put a credential on the mini, the USER types it via `read -s` directly in
  the mini Terminal.** The agent never carries, echoes, or logs it.
- **Never type a secret char-by-char through the canvas** (shift-map or
  otherwise) — it'd be visible in screenshots and reasoning.
- For mini-local work needing a secret, delegate to mini Claude, which can read
  it from the mini's Keychain / 1Password locally.

## Out of scope (v1) / v2

- Right-click (no CDP path on macOS — menu-bar substitute).
- AppleScript automation (was gist-delivered; removed with gist). A genuine need
  is a separate opt-in channel, not this skill.
- Unicode / emoji / bulk paste — clipboard or file channel (v2). The
  "Synchronize clipboard" toggle being ON does not bridge `pbcopy`->`Cmd+V` under
  CDP (needs a real OS-foreground gesture — same root cause `/windows` hit). A
  macOS `Cmd+V` from a mini-locally-set clipboard is a v2 candidate.

## History appendix — why this changed (harvested from the gist-era INCIDENTS)

This is diagnostic gold: it records the **exact counter-evidence** to the current
direct-CDP premise, so a future agent re-checks rather than blindly trusting
either side. Full originals are in `archive-gist-era/INCIDENTS.md`,
`archive-gist-era/HARDWARE-FINDINGS-2026-04-27.md`, and
`archive-gist-era/AGENT-GUIDE.md`.

- **2026-04-30 — `click_at` VALIDATED through the CRD canvas.** With
  chrome-devtools-mcp's `--experimental-vision` flag, `click_at(x,y)` was tested
  end-to-end into `plan2bid-minim4`: clicks forwarded (~50ms), focused the LEFT
  Terminal, a `type_text("date")` landed there, and a mini-Claude conversation
  round-tripped. A ~50px Finder icon double-click landed but missed center
  (small-target precision caveat — still UNVERIFIED today).
- **2026-05-19 — `click_at` DEPRECATED -> gist/cliclick promoted.** The
  `--experimental-vision` channel went unreliable upstream (the MCP server
  wedged on certain sequences, needing the `/devtools` kill + npx-hash scrub to
  recover). The team concluded any synthetic dev-side click depends on CRD not
  enforcing `isTrusted`, judged that an architectural dead end, and re-routed all
  clicks through `cliclick` on the mini via `gh gist` transport, with per-mini
  `/macmini measure` calibration. THIS is the architecture the 2026-06-01 rewrite
  reverses.
- **2026-06-01 — direct CDP RE-CONFIRMED, clean cut.** Live test (this doc's
  Verified section) shows `click_at` reaches the host again — on the **standard**
  MCP config, **without** `--experimental-vision`. Most likely the 2026-05-19
  break was specific to that beta flag (now avoided), or a since-resolved
  CRD/Chrome change. The user wants GitHub gone; the verified-working direct path
  argues for the clean cut. The skill carries the first-session smoke tests
  (Apple-menu click + shift-map + Cmd+Space) so an agent re-checks the premise
  each session instead of trusting it blind.

**The diagnostic rule:** if `click_at` stops reaching the host in some future
session, FIRST check whether `--experimental-vision` got re-added to the MCP args
(that's the historical instability trigger). There is no on-host fallback in the
new skill — a broken click channel is STOP-and-escalate, not a silent reroute.

## Other still-true macOS-substrate facts (harvested)

- **CRD strips Shift on outbound keystrokes** (Chromium issues 40355503 /
  40933947) — `type_text` arrives lowercased/unshifted. The `press_key` shift-map
  is the fix (verified this session). This is why credentials/arbitrary capitals
  never go through `type_text`.
- **Programmatic clipboard sync (dev -> mini) does not trigger CRD's onPaste** —
  it requires a real user gesture; CDP-injected events don't qualify. So
  `pbcopy`->`Cmd+V` pastes the mini's LOCAL clipboard, not dev's. Clipboard
  bridge stays v2.
- **CRD's a11y tree is stripped to `ignored`** for its panel controls (Begin,
  Synchronize clipboard, Send system keys); only the `Desktop` textbox wrapper is
  exposed. `document.querySelectorAll('[role="checkbox"]')` still finds them in
  the DOM (no shadow piercing) for coordinate mapping, but `click({uid})` can't
  reach them — hence the coordinate/user fallback for LAYER-1.
- **CRD PIN / Google sign-in are intentionally user-only** — the agent lands the
  user on the page and hands off; never types/reads/stores the PIN.
- **macOS permission dialogs** (Gatekeeper / "app wants to control your computer"
  / Accessibility / Screen Recording) can appear (esp. first Spotlight launch)
  and may hide BEHIND the foreground app — surface to the user; don't
  blind-approve.
